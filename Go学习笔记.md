### 五大核心命令
1. go run 文件名.go
作用：临时运行源码，不生成任何可执行文件
示例：go run main.go
结果：终端直接输出运行结果，目录无新增文件
使用场景：代码实时调试、快速测试效果
2. go build 文件名.go
作用：编译源码，生成独立可执行程序
Windows：执行后生成main.exe，双击即可运行，无需安装 Go 环境
Mac/Linux：生成无后缀执行文件，命令./main运行
使用场景：项目打包、程序分发交付
3. go mod init 项目名
作用：初始化 Go 项目，生成依赖配置文件go.mod
示例：go mod init mygo
结果：自动创建go.mod，记录项目名称、Go 版本
使用规则：新建 Go 项目第一条必执行命令，项目名可自定义
4. go mod tidy
作用：自动下载缺失第三方依赖，清理无效废弃依赖
使用场景：代码引入外部开源包、拉取他人项目、删减无用依赖
效果：联网补齐依赖包，生成go.sum校验文件，消除代码爆红
1. go env
作用：查看 Go 全部环境配置信息
重点参数
GOROOT：Go SDK 安装路径
GOPATH：旧版项目工作目录
用途：排查环境异常、配置出错问题

# Go byte 与 rune 完整笔记
## 一、Go 两种字符类型
### 1. byte（uint8）【占一个字节】
1. 底层等价：`type byte uint8`，8位无符号整数，范围 0~255
2. 作用：存储**单个ASCII字符**（英文、数字、英文标点、空格）
3. 特点：1个ASCII字符 = 1个byte；中文UTF-8占3个byte，会被拆成多段byte碎片
4. 适用场景：文件读写、网络二进制流、纯英文文本，占用内存小、性能高

### 2. rune（int32）【占四个字节】
1. 底层等价：`type rune int32`，32位整数
2. 作用：存储**单个Unicode字符**（中文、日文、韩文、emoji、特殊符号）
3. 特点：无论中英文，1个肉眼可见字符 = 1个rune，自动合并UTF-8多字节碎片
4. 适用场景：含中文文本、统计文字个数、截取汉字、遍历单个字符

## 二、字符串底层规则
1. Go 字符串底层存储结构是 `[]byte`（字节切片）
2. 编码默认 UTF-8：
   - 英文字符：1字节
   - 中文字符：3字节
3. 长度区分（关键考点）
```go
s := "你好 golang"
// len(s)：获取byte字节总数（底层存储碎片数量）
fmt.Println(len(s)) // 3+3+1+6 = 13
// []rune(s)：转换为完整字符切片，长度=肉眼看到的文字数量
fmt.Println(len([]rune(s))) // 2汉字+1空格+6字母 = 9
中文占用字符多，统一用rune遍历
字符串不可直接修改内容
不同类型不能直接运算，必须转成同类型
**字符串不能直接修改原因：**
Go 中字符串是只读常量，底层字节数组不可变
一旦定义好字符串，内存里内容固定，不允许原地改字符
想修改只能：
先转成切片 → 修改 → 再转回字符串
```
运行
// 错误写法
s := "abc"
s[0] = 'd' // 直接报错

// 正确修改方式
arr := []rune(s)
arr[0] = 'd'
s = string(arr)
```
底层原理：修改字符串本质是新建新字符串，不是改动原数据
中文用 rune 遍历原因
编码不同
英文 / 数字：1 个字符占1 字节，用byte足够
中文：1 个汉字占3 字节，byte存不下
遍历结果差异
byte遍历中文：乱码、拆分残缺
rune专门适配UTF-8中文，遍历正常汉字
口诀：英文用 byte，中文必用 rune

# Go结构体精简笔记
## 1. 结构体 struct
- 作用：封装多类型数据，Go面向对象基础
- 数据类型：值类型，存储在栈，声明自动填充对应零值
- 定义格式：`type 结构体名 struct {字段名 数据类型}`

## 2. type 两种用法
1. 自定义类型：`type T int`
   全新独立类型，支持绑定方法，赋值需要强制类型转换
2. 类型别名：`type T = int`
   仅别名，不会生成新类型，无法绑定方法，变量可直接赋值

## 3. 结构体3种初始化方式
1. 逐个字段赋值：灵活，未赋值字段保留零值
2. 按顺序赋值：必须严格匹配字段顺序、字段总数
3. 指定字段赋值（开发推荐）：顺序不限，支持局部赋值，兼容性高
- 补充：结构体互相赋值为完整值拷贝，两份数据互相独立

## 4. 方法 & 接收者
1. 值接收者：拷贝结构体数据，仅可读，修改不会作用原数据
2. 指针接收者：直接操作原结构体，修改永久生效，节省大数据拷贝开销
3. 语法糖：结构体变量可直接调用指针接收方法，无需手动写`&`
4. 编码规范：同一个结构体所有方法统一使用指针接收者

## 5. 自定义类型添加方法
- 硬性规则：int/string/slice/map原生类型不能绑定方法，仅`type 新类型 原类型`自定义类型可用
- map特殊点：`[]`下标无自动解引用语法糖，指针接收者操作map必须手写`(*m)`

## 6. 基础类型匿名字段
- 不写字段名，默认字段名 = 数据类型名
- 同一结构体不能存在两个同类型匿名字段
- 自动继承该类型所有方法

## 7. 具名嵌套结构体
- 将结构体作为带名字段嵌入
- 访问格式：`外层变量.字段名.内层字段`，不能简写访问

## 8. 匿名嵌套结构体
- 直接写入结构体类型，无自定义字段名
- 内嵌的字段、方法自动提升，外层可直接调用
- Go模拟继承的核心语法

## 9. 名字冲突处理规则
1. 就近优先：结构体自身字段优先级 > 内嵌字段；读取内嵌重名字段需要标明结构体名称
2. 嵌入多个结构体出现重名字段/方法：简写调用报错，必须指定结构体访问

## 10. 匿名嵌套模拟继承
- Go无原生继承，依靠组合（匿名嵌套）实现字段、方法复用
- 被嵌入结构体：父结构体（存放公共属性、公共方法）
- 外层结构体：子结构体，可扩展独有字段、重写父类方法
- 重写后想要调用父类方法，需要手动指定结构体调用
- 本质是组合关系，耦合度更低，代码更好维护

## 速记口诀
结构体存数据，type分清类型别名；
三种初始化写法，方法区分两种接收；
原生类型不能绑方法，map指针记得解引用；
具名嵌套分层访问，匿名嵌套直接提升；
重名就近优先，多内嵌重名必须指明；
摒弃原生继承，组合复用为最优方案。
